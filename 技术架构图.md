## 3. LangGraph工作流示例：智能客服处理流程

```mermaid
stateDiagram-v2
    [*] --> 接收请求
    
    state 接收请求 {
        [*] --> 解析意图
        解析意图 --> 意图分类
        意图分类 --> 闲聊?:是否存在明确业务意图?
        
        闲聊?:是否存在明确业务意图?
        闲聊?: --> 是: 有业务意图
        闲聊?: --> 否: 闲聊/寒暄
    }
    
    闲聊 --> 生成闲聊回复
    
    state 业务处理 {
        生成闲聊回复 --> [*]
        
        是 --> 识别实体
        识别实体 --> 查询知识库
        查询知识库 --> 生成答复
        生成答复 --> 引用验证
        引用验证 --> 质量检查
        质量检查 --> [*]
    }
    
    state 质量检查 {
        [*] --> 风险词检测
        风险词检测 --> 敏感内容?
        
        敏感内容? --> 是: 人工审核
        敏感内容? --> 否: 直接回复
        人工审核 --> 修改答复
        
        修改答复 --> [*]
        直接回复 --> [*]
    }
```

## 8. 智能体消息协议图

```mermaid
sequenceDiagram
    participant User as 用户
    participant Master as 主控智能体
    participant Product as 商品智能体
    participant Hermes as Hermes消息总线
    participant Risk as 风控智能体
    
    User->>Master: 用户请求（多语言）
    Master->>Master: 意图解析 & 任务分解
    
    par 并行执行
        Master->>Product: TaskMessage(商品查询)
        Product-->>Hermes: ResultMessage(商品信息)
    and 并行执行
        Master->>Risk: TaskMessage(风险评估)
        Risk-->>Hermes: ResultMessage(风险分数)
    end
    
    Hermes-->>Master: 汇总结果
    Master->>Master: 结果整合 & 策略生成
    
    alt 高风险操作
        Master->>Risk: QueryMessage(人工复核)
        Risk-->>Master: 复核结果
    end
    
    Master->>User: 智能响应
```
